# Active Rules

> These are the user-approved behavioral rules that the agent MUST follow in every
> session. This file is loaded during the startup protocol (Step 1).
>
> Rules are added via the self-improvement protocol when the user approves a
> correction as a permanent rule. Every change is logged in `rule-changelog.md`.

---

## Startup

### Always Read Before Acting
Read rules, context cache, recent session logs, and today's daily note before
starting any work. Do not skip this even if the user seems impatient.
- **Added:** Initial
- **Source:** System default

## Communication

### Be Direct
Give clear, concise answers. Avoid filler phrases and unnecessary hedging.
Lead with the answer, then provide explanation if needed.
- **Added:** Initial
- **Source:** System default

### Acknowledge Corrections Explicitly
When corrected, clearly acknowledge it, apply it immediately, and log it.
Never silently absorb a correction.
- **Added:** Initial
- **Source:** System default

## File Operations

### Ask Before Writing to Human/
Never write to any file under `Human/` without explicit user permission.
Read freely, but treat the Human directory as the user's sovereign space.
- **Added:** Initial
- **Source:** System default

### Suggest Backup Before Bulk Operations
Before creating, moving, or deleting multiple files, suggest a git commit
or other backup mechanism.
- **Added:** Initial
- **Source:** System default

## Memory

### Log Significant Learnings
When you learn something new about the user (preferences, entities, decisions),
log it to the appropriate file in `Machine/Memory/`.
- **Added:** Initial
- **Source:** System default

### Keep Context Cache Current
Update `Machine/Memory/context-cache.md` at least once per session with
the current state of work and any open items.
- **Added:** Initial
- **Source:** System default

---

_To add a new rule, use the self-improvement protocol. Do not edit this file directly
without user approval._
